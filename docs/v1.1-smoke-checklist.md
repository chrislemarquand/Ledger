# v1.1 Manual Smoke Checklist

Work through each item in order. Call out by number with Pass / Fail + any notes.

---

## 1. About Panel

**1.1** Open About Ledger (app menu → About Ledger). Version line should read **Version 1.1 (NNN)** — build number appears once only, not twice.

---

## 2. Settings Window

**2.1** Press **Cmd+,** — Settings window opens.
**2.2** Close it. Open via app menu → **Settings…** — opens again.
**2.3** Resize main window to something small. Open Settings — window should clamp to visible screen height, not overflow.
**2.4** In **General** pane — pane should auto-size, no hard-coded blank space.
**2.5** In **Inspector** pane — scroll view starts at the top (first item visible, not scrolled mid-way).
**2.6** Toggle a field visibility checkbox — only that row updates, no full pane flicker/rebuild.

---

## 3. Toolbar & Appearance

**3.1** Switch macOS between Light and Dark mode while Ledger is open. Toolbar buttons should update immediately without requiring a restart or window move.
**3.2** Confirm the **Export** toolbar button shows a `square.and.arrow.up` icon in the File menu.

---

## 4. Sort Order Consistency

**4.1** Open the sort menu from the toolbar. Note the order of the four sort options (Name / Date / Size / Kind).
**4.2** Open **View** menu → sort submenu. Confirm options appear in the same order.

---

## 5. Export — Scope Picker

**5.1** Select 1+ files. Click **File → Export → Create CSV…** (or toolbar). A sheet appears asking **Selection (N files)** or **Folder** — not immediate action.
**5.2** Choose Folder. A save-panel sheet slides down (not a floating modal).
**5.3** With staged (unapplied) metadata edits present, click **Send to Photos…**. Sheet warns about pending edits before proceeding.
**5.4** With staged edits, click **Send to Lightroom Classic…**. Same warning.
**5.5** With nothing selected, confirm **Selection** option is greyed out or absent in the scope picker.
**5.6** Confirm **Create CSV…**, **Send to Photos…**, **Send to Lightroom Classic…** all have ellipsis (…) in the menu.
**5.7** Open context menu on a gallery or list item — confirm **Send to Photos** and **Send to Lightroom Classic** are **not** present in context menus.

---

## 6. Inspector Status Box

**6.1** Apply a metadata change to a file. Confirm there is **no "Last Operation" box** at the bottom of the inspector. Result is reflected in the field values only.

---

## 7. Import — General Sheet Behaviour

**7.1** Open import sheet via File menu. Confirm all source type names read: **CSV**, **GPX**, **Reference Folder**, **Reference Image**, **EOS-1V** (hyphen, not space).
**7.2** Click the **ⓘ** icon next to each source type title. Popover appears, text is fully visible (not clipped), popover resizes to content.
**7.3** With fewer than 2 files selected, confirm import scope defaults to **Folder** (not Selection).
**7.4** Change scope to Selection, then clear the selection. Scope should revert to Folder automatically. *(Note: revert happens when sheet reopens, not dynamically mid-session — this is expected.)*
**7.5** Open an import sheet, change some options, close without importing, reopen — **selectedTagIDs** reset (no fields pre-checked from last session).

---

## 8. Import — CSV

**8.1** Import a CSV file matching by row order. Row labels in preview show **Row 001**, **Row 002** etc. (not filenames for row-order match).
**8.2** If row-order fallback is used, a warning banner appears in the sheet (not hidden in popover only).
**8.3** Run a clean import (all rows matched, no warnings). Sheet auto-closes after apply.
**8.4** Run an import that produces warnings or conflicts. Sheet stays open showing post-import review mode with a single banner. **Import** button becomes **Close**. **Fields** and **Advanced** tabs are disabled. **Details…** button shows structured report.

---

## 9. Import — ExifTool CSV

**9.1** Export a CSV via **File → Export → Create CSV…**, then immediately re-import it. Warning count should be low (ExifTool-origin files don't generate spurious column warnings).
**9.2** Preview shows accurate field-write count.

---

## 10. Import — GPX

**10.1** Select a GPX file. Source picker allows selecting **multiple GPX files** if needed.
**10.2** Clean import auto-closes. Import with unmatched files stays in review mode.

---

## 11. Import — Reference Folder / Reference Image

**11.1** Reference Folder: select a folder of reference images. Preview shows matched files.
**11.2** Reference Image: select a single image. Preview shows one-to-many apply across scope.
**11.3** GPS coordinates imported from reference: sign normalised correctly (no accidental negation).

---

## 12. Import — EOS-1V

**12.1** Import an EOS-1V CSV. Sheet title and all labels read **EOS-1V** (hyphenated).
**12.2** If multiple lenses match a focal length: deconfliction dialog appears with title **"Multiple Lenses Matched"**, shows the target filename in body text, has **Use Lens** and **Cancel** buttons (stacked, not side by side), and an **"Apply to N more images"** checkbox.
**12.3** Focal Length field: if **Focal Length** is deselected, **Lens Model** field greys out (dependent field).
**12.4** EOS-1V always uses unlimited row count (no row-count cap applied).
**12.5** Clean import auto-closes; partial/conflict import stays in review mode with Details… report.

---

## 13. Import — Banners

**13.1** Trigger a scope-mismatch or fallback warning. Banner appears as plain secondary text (no coloured/tinted background box).
**13.2** Only one banner is shown at a time (highest-priority wins).
**13.3** Banners use plain language (no jargon, no raw tag names in user-facing copy).

---

## 14. Apply Flow

**14.1** Stage edits across multiple files. Click **Apply**. Single confirmation sheet (not a floating alert). Confirms correctly.
**14.2** Confirm no `runModal` blocking — sheet is attached to window.
**14.3** After apply, inspector preview updates to reflect new orientation/values (no stale pre-apply image shown while thumbnail already updated).

---

## 15. Restore

**15.1** After an apply, use **Image → Restore from Backup**. Restore works. Subtitle shows **"Restored X of N — Y failed"** format if partial.

---

## 16. Pluralisation

**16.1** With 1 file selected, any status messages or confirmations read "1 file" (not "1 files").
**16.2** With 2+ files, reads "N files" correctly.
**16.3** Spot-check "backup" / "image" plurals in any visible UI strings.

---

## 17. Stability Spot-Check

**17.1** Open a large folder (100+ images). Browse freely. No freeze, no crash, thumbnail cache loads progressively.
**17.2** Switch folders rapidly several times. No ghost images, no stale browser state.
**17.3** Open a preset editor sheet, close without saving, reopen — no stale validation message or duplicate-conflict state carried over.
**17.4** Open preset manager, mark a preset for delete, close, reopen — no stale pending-delete state.

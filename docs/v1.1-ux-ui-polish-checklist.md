# v1.1 UX/UI Polish Checklist

Purpose: final UX/UI polish pass before shipping v1.1.

## Import / Export UX

- [ ] Export scope explicitness (`X1`)
  - Make export intent explicit before confirm: `Folder (N files)` vs `Selection (N files)`.
  - Ensure the same wording appears consistently in sheet copy and any confirmation/status messaging.

- [ ] CSV warning quality for ExifTool-origin files (`X4`)
  - Reduce warning noise in preview.
  - Prioritize a concise summary (grouped warning categories and counts) before detailed per-row warnings.
  - Keep detailed warnings accessible for troubleshooting.

- [ ] Lens deconfliction dialog layout (`E6`)
  - Polish spacing, hierarchy, and readability of the ambiguous-lens prompt.
  - Ensure candidate selection and the "apply to remaining rows at this focal length" option are visually clear and scannable.

- [ ] Preview text legibility + warning placement
  - Improve readability/scannability of preview text content (especially long warning-heavy previews).
  - Evaluate moving key warnings from popover preview into the main import sheet so critical issues are visible without opening preview.

- [ ] Status surface rationalisation (inspector/status bar/dialogs)
  - Remove the bottom-of-inspector status box.
  - Define one clear rule-set for where state information appears (inspector inline, status bar, or modal/dialog).
  - Eliminate duplicate/conflicting status messaging across those surfaces.
